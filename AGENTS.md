# AGENTS.md

## Project Overview

ziyuanzhan is a Python-based monitoring tool that automatically checks the availability of video resource sites listed on ziyuanzu.com. It runs on GitHub Actions (every 2 hours), performs HTTP health checks, and generates a dark-themed status dashboard deployed via GitHub Pages.

## Architecture

- `monitor.py` - Core script: fetches site list from ziyuanzu.com, runs HTTP health checks, generates `docs/index.html` and `data/latest.json`
- `mcp_server.py` - MCP (Model Context Protocol) server providing 6 query tools for AI assistants
- `.github/workflows/monitor.yml` - GitHub Actions cron workflow (every 2 hours)
- `docs/index.html` - Auto-generated monitoring dashboard (GitHub Pages source)
- `data/` - JSON data files (latest + historical archives)

## Tech Stack

- Python 3.11+
- requests, beautifulsoup4, lxml (web scraping)
- mcp (Model Context Protocol server)
- GitHub Actions (CI/CD scheduling)
- GitHub Pages (static hosting)

## Always Do

- Read `monitor.py` and `mcp_server.py` to understand the data flow before making changes
- Ensure any changes to `monitor.py` maintain backward compatibility with the JSON data schema in `data/latest.json`
- Test locally with `python monitor.py` before pushing changes
- Keep the generated `docs/index.html` self-contained (inline CSS, no external dependencies)

## Ask First

- Changing the cron schedule frequency
- Modifying the data source URL (ziyuanzu.com)
- Adding new MCP tools (coordinate with existing tool naming convention)

## Never Do

- Commit manually generated `docs/index.html` or `data/*.json` files (they are auto-generated by Actions)
- Add dependencies not listed in `requirements.txt`
- Remove or rename existing MCP tools without deprecation notice
